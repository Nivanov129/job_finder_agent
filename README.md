# Job Agent

Локальный агент подбора вакансий с **веб-интерфейсом**. Собирает вакансии из
Telegram-каналов и агрегаторов, скорит под резюме пользователя его же AI
(bring-your-own), показывает подборку в UI и отдаёт `.xlsx`. Работает на машине
пользователя; данные не покидают её, кроме как к выбранному AI-движку и
self-host web-поиску.

Ценность сверх hh: оценивает не только «подхожу ли» (**резюме %**), но и «хочу ли»
(**карта %**), готовит черновик сопроводительного, который заходит с твоих
результатов, и ищет контакты под отклик. Каждый ставит у себя — ничего не
хостится централизованно, ключи не раздаются.

> **Что именно работает сейчас** — см. [`docs/FEATURES.md`](docs/FEATURES.md):
> экраны веб-UI (Агент, Подбор за период, Подборка с ползунком по карте, Поиск
> контактов, Настройка, AI, Telegram), стадии пайплайна, контакты и инвестигатор.

## Что внутри

- **Веб-интерфейс** (`http://localhost:8766`): дашборд агента, разовый «Подбор
  за период» с живой воронкой, «Подборка» с двойными кольцами и ползунком по
  карте, «Поиск контактов» по ссылке/PDF, настройка и авторизация AI.
- **Bring-your-own AI.** Подключаете свой Claude/Codex (CLI на подписке), ключ
  Anthropic/OpenAI или локальный Ollama. Продукт бесплатный.
- **Направления (`tracks[]`).** От одного до N: у каждого своё резюме, карта
  поиска, шаблон сопроводительного и **«Допустимые роли», которые генерятся из
  резюме**. При единственном треке UI и выхлоп схлопываются.
- **Без автооткликов и авто-DM.** Контакт-модуль готовит черновик — отправка
  руками.

> **Не технарь?** Отдай файл [`llms.md`](llms.md) любой нейросети (ChatGPT/Claude)
> и напиши «помоги установить» — она проведёт по шагам без терминала-страха.

Архитектура, инварианты и порядок задач — в
[`docs/plans/completed/job-agent-plan.md`](docs/plans/completed/job-agent-plan.md); конвенции проекта —
в [`CLAUDE.md`](CLAUDE.md).

## Онбординг (по шагам)

### 1. Docker

Нужен Docker Desktop (или Docker Engine + Compose v2 на Linux). Установка самого
Docker — на вас: <https://www.docker.com/products/docker-desktop/>. Установщик
проверит наличие Docker и демона, но ставить его не будет.

Запустите тонкий установщик из корня репозитория:

```sh
./install.sh          # macOS / Linux
```

```powershell
.\install.ps1         # Windows (PowerShell)
```

Скрипт ([`install.sh`](install.sh) / [`install.ps1`](install.ps1)) проверяет
Docker, создаёт каталог данных (`./data` по умолчанию, переопределяется
`JOB_AGENT_DATA`), кладёт туда `config.json` из [`config.example.json`](config.example.json)
при первом запуске (повторный запуск его не перетирает), собирает образы по
[`Dockerfile`](Dockerfile), поднимает стек из [`compose.yml`](compose.yml) и
открывает конфиг в редакторе.

Стек: пайплайн + self-host SearXNG (web-поиск) + одноразовый прогрев
мультиязычной модели эмбеддингов `multilingual-e5-large` в общий том.

### 2. Конфиг

Все настройки — в одном `config.json` (в каталоге данных), валидном по
[`config.schema.json`](config.schema.json). Заполните:

- **`tracks[]`** — направления. Для каждого: `id` (латиница/цифры/`-`/`_`),
  `name` (человеческий ярлык — идёт в выхлоп как «направление»), `resume_path`
  (резюме под направление), опц. `cover_template_path` (без него сопроводительное
  для трека не генерится), `rubric`, `role_gate`, `disqualifiers`. Минимум один трек.
- **`search_map`** — карта поиска: `path` к файлу примеров-идеалов и/или
  структурированные `examples`. Пример с `track_id` считает «карту %» против
  своего направления; без `track_id` — общий для всех.
- **`global_role_gate` / `global_disqualifiers`** — дефолтный гейт по роли и
  красные флаги; трек может переопределить/дополнить.
- **`tg_channels[]`** — каналы (`handle` без `@`, `private` для приватных) и
  **`use_aggregators`** — vseti.app + getmatch.
- **`output_mode`** (`table` / `bot` / `both`), **`backfill_days`** (глубина
  первого прогона), **`cover_letter_threshold`** (мин. overall % для
  сопроводительного, дефолт 70), **`output_lang`** (`ru`/`en`).
- **`enable_contacts`** (дефолт выкл) — контакт-ассист.

Резюме (`resume_path`), шаблоны (`cover_template_path`) и файл карты
(`search_map.path`) положите в каталог данных и укажите пути относительно него
(в контейнере он смонтирован как `/data`). Образец шаблона сопроводительного —
[`prompts/default.md`](prompts/default.md).

### 3. Движок AI (`scoring_engine`)

| Значение | Что нужно | Заметки |
|---|---|---|
| `cli` (дефолт) | Установленный `claude` или `codex`, указать `cli_tool` | Жжёт агентный лимит подписки |
| `api_key` | `api_base_url` + `api_key` (Anthropic/OpenAI) | Не лейте резюме в тарифы, обучающиеся на промтах — см. «Безопасность» |
| `ollama` | Запущенный Ollama + `ollama_model` | Полностью локально |

Переключение между движками — только правка конфига, без изменения кода.

### 4. Web-поиск (`web_search`)

Нужен для анализа компании на скоринге и для контакт-поиска.

- **`searxng`** (дефолт) — self-host, поднимается в составе стека. Внутри
  compose-сети укажите `web_search.url = http://searxng:8080` (имя сервиса, не
  `localhost`). Порт `127.0.0.1:8080` открыт только для отладки/калибровки.
- **`serp`** — внешний SERP-провайдер по `web_search.api_key`.

### 5. Telegram-бот (`output_mode` с `bot`)

1. Создайте бота через [@BotFather](https://t.me/BotFather) → получите токен.
2. Положите токен в `bot_token` (или, как секрет, в `.env` — см. ниже).
3. Напишите боту первым (`/start`), чтобы он мог писать вам в личку. Бот шлёт
   карточки только в личный чат владельца, без авто-DM кому-либо.

### 6. Приватные каналы (Telethon, опц.)

Только если в `tg_channels[]` есть `"private": true`. Заполните `telethon_creds`
(`api_id`, `api_hash` с <https://my.telegram.org>, строка `session`). Без
`telethon_creds` приватные каналы пропускаются, публичные (`t.me/s/<handle>`)
работают всегда.

### 7. Секреты через `.env`

Ключи API, токен бота и Telethon-creds держите в `.env` рядом с `compose.yml`
(он подхватывается, если файл есть). В образ и в логи секреты не попадают.
Никогда не коммитьте `.env` и не хардкодьте ключи.

### 8. Backfill → проверить xlsx

Первый исторический прогон за N дней (стадии сбор → нормализация → дедуп →
пре-фильтр/роутинг → скоринг → обогащение → xlsx):

```sh
docker compose run --rm pipeline \
  backfill --days 14 --config /data/config.json --out /data/job-agent-result.xlsx
```

В логах: «собрано N · после фильтра M · топ-K». Откройте
`./data/job-agent-result.xlsx` — колонки `дата · источник · должность · компания ·
ссылка/контакт · зарплата · резюме % · карта % · направление · вердикт · гэпы ·
сопроводительное · контакты`. Бейджи процентов залиты цветом по диапазону;
колонка `направление` скрыта при единственном треке.

Калибровочный флаг `--min-sim` задаёт порог близости пре-фильтра (по умолчанию —
`min_sim` из конфига, мягкий старт `0.30`). Подобрать порог под свои данные:

```sh
docker compose run --rm pipeline calibrate --days 14 --config /data/config.json
```

Команда `calibrate` прогоняет сбор → нормализацию → роутинг без скоринга, печатает
распределение `best_sim` и рекомендованный порог (метод Оцу — естественная граница
между релевантной кучей и шумом). Запишите выбранное число в `config.json` как
`"min_sim"`.

### 9. Ночной режим (always-on)

После того как backfill подтвердил пайплайн, оставьте стек поднятым — cron в
контейнере раз в сутки гоняет инкрементальный прогон (`nightly`, только новое с
прошлого раза, seen-store как водяной знак):

```sh
docker compose up -d            # поднять стек в фоне
docker compose logs -f pipeline # смотреть прогоны
docker compose down             # остановить
```

> **Always-on:** ночной мониторинг работает, только пока хост включён. На спящей
> или выключенной машине прогона не будет — пропущенное добирается ручным backfill.

Вне Docker встроенный цикл-планировщик: `job-agent nightly --serve --at 03:00`.

## Безопасность и риски

- **Данные не покидают машину**, кроме как к выбранному вами AI-движку и
  web-поиску. Никаких CDN, никакой телеметрии.
- **Резюме и API-тарифы.** Если используете `api_key`, предпочтите локаль/CLI
  или провайдера с режимом no-training — не отдавайте резюме в тарифы,
  обучающиеся на промтах.
- **Бан аккаунтов.** Никаких автооткликов и авто-DM. Бот пишет только владельцу
  в личку. Telethon — на ваш аккаунт и ваш риск.
- **Хрупкость скрапа.** Вёрстка агрегаторов меняется; логика изолирована в
  адаптерах — поломка одного источника не валит весь прогон.
- **Секреты** — только через env/`.env`, не в конфиг-репозиторий, не в логи.

## Разработка

Требуется [`uv`](https://docs.astral.sh/uv/).

```sh
uv sync                       # установить зависимости
uv run ruff check src tests   # линт
uv run pytest -q              # тесты
```

Юнит-тесты никогда не ходят в сеть: внешние границы (AI-движок, web-поиск,
Telegram, HTTP-сбор, эмбеддинги) — за интерфейсами, в тестах фейки/фикстуры.
Визуальный язык — [`design/design-tokens.md`](design/design-tokens.md)
(прототип: [`design/job-agent-prototype.html`](design/job-agent-prototype.html)).
Промты-контракты — в [`prompts/`](prompts/).
