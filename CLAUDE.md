# CLAUDE.md — Job Agent

Конвенции и инварианты проекта. Этот файл читается в начале каждой свежей сессии — держим его коротким и обязательным к исполнению.

## Стек и команды

- Python 3.12, менеджер пакетов `uv`. Линт/формат `ruff`, тесты `pytest`, модели `pydantic` v2, типы `mypy` (нестрогий, не блокер).
- Перед отметкой задачи выполненной обязаны проходить: `uv run ruff check src tests` и `uv run pytest -q`.

## Тесты

- **Юнит-тесты никогда не ходят в сеть.** Все внешние границы (AI-движок, web-поиск, Telegram, HTTP-сбор, модель эмбеддингов) — за интерфейсами; в тестах фейки/фикстуры. HTTP-фикстуры лежат в `tests/fixtures/`.
- Каждая стадия пайплайна — модуль с явным интерфейсом; `pipeline.py` сшивает их, а не инлайнит логику.

## Инварианты архитектуры (не нарушать)

- **Направления — динамический список `tracks[]` (1..N).** Никогда не хардкодить «два пути» / «Скейлап» / «AI-first» в коде или UI. UI использует повторяемую карточку направления. При единственном треке теги направления скрываются.
- **Всё кодирование цветом — из `src/job_agent/presentation.py`.** Бейдж-диапазоны процентов и маппинг иконок вердикта берут оттуда и xlsx, и бот, и web-UI. Не дублировать цвета.
- **Продукт не делает внешних сетевых вызовов**, кроме как к выбранному пользователем AI-движку и настроенному web-поиску. Никаких CDN (иконки Tabler вшивать локально), никакой телеметрии.
- Конфиг валидируется против `config.schema.json`. Секреты только через env/`.env`, никогда не хардкодить и не логировать.

## Контракты (read-only входы)

`prompts/*.md`, `design/*`, `config.schema.json` задают контракты — использовать как есть, не переписывать, если задача явно не просит.
