# Дизайн — источник истины

Извлечено из `design/prototype.html`. Визуальный язык берём из прототипа дословно; **структуру обобщаем под `tracks[]`** (прототип показывает старую захардкоженную пару путей — см. «Расхождение» ниже).

## Токены (CSS-переменные)

```css
--font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;

/* поверхности */
--surface-0:#f5f4ef;  --surface-1:#faf9f5;  --surface-2:#ffffff;

/* текст */
--text-primary:#1a1a18;  --text-secondary:#5f5e5a;  --text-muted:#8a8980;

/* бордеры */
--border:rgba(0,0,0,.10);  --border-strong:rgba(0,0,0,.18);

/* акцент */
--text-accent:#185FA5;  --bg-accent:#E6F1FB;  --border-accent:#378ADD;
--fill-accent:#2B7FD4;  --on-accent:#ffffff;

/* предупреждение */
--text-warning:#854F0B;  --bg-warning:#FAEEDA;  --border-warning:#EF9F27;

/* успех */
--text-success:#3B6D11;

--radius:8px;            /* кнопки/инпуты; карточки 12px; внутренние панели 10px */
```

Принципы: плоско, светлые поверхности, бордеры 0.5px, скругление карточек 12px, один акцентный цвет, без теней и градиентов. Sentence case. Бейдж-процент — единственное цветовое кодирование (по нему сразу читается «стоит ли»).

## Бейдж «резюме %» (цвет по диапазону) — общая спека для xlsx, бота и web-UI

| Диапазон | Фон | Текст | Смысл |
|---|---|---|---|
| зелёный, overall ≥ 80 | `#EAF3DE` | `#27500A` | сильное соответствие |
| янтарный, 70–79 | `#FAEEDA` | `#633806` | средне, на дотяг |
| серый, < 70 | `#F1EFE8` | `#444441` | слабо |

Пороги диапазонов конфигурируемы; значения выше — дефолт из прототипа. «Карта %» бейджем не красится — идёт строкой с иконкой `ti-map-2`.

Тег направления (бейдж): фон `#EEEDFE`, текст `#3C3489`. **Скрыт при единственном треке.**

## Вердикт (иконка + тон) — по `verdict.type` из ScoreResult

| Вердикт | Иконка (Tabler) | Цвет | type |
|---|---|---|---|
| Точное попадание · откликаться | `ti-circle-check` | `--text-success` | `precise_fit` |
| Stretch · стоит откликнуться | `ti-arrow-up-right` | `--text-warning` | `stretch` |
| На грани · решать по приоритету | `ti-minus` | `--text-muted` | (overall ниже зоны) |

## Иконки

Tabler Icons. В прототипе подключены с CDN (`cdn.jsdelivr.net`) — **в продукте вшить локально** (принцип «всё локально, без внешних обращений»): положить webfont в статику web-UI, не тянуть с CDN.

## Инвентарь компонентов

**Экран 1 · Настройка** (один прокручиваемый столбец, max-width 720px):
- Шапка приложения: иконка `ti-radar-2` в акцентном квадрате + «Job agent · настройка» / «локально на вашем компьютере».
- Warning-плашка (`bg-warning`): про always-on.
- Карта «Профиль»: иконка `ti-user`. Внутри — **список карточек направлений** (см. расхождение), каждая: бейдж-имя направления + строка файла резюме (иконка `ti-file-cv`/`ti-file-text`, имя файла, `ti-check` при загруженном) + строка «Сопроводительное · пример» (`ti-mail`, «загрузить»). Снизу общий пунктирный блок «Карта поиска · общая» (`ti-map-2`).
- Карта «Источники»: `ti-rss`. Textarea для каналов (плейсхолдер `@ml_jobs`…) + чипы агрегаторов `vseti.app`/`getmatch` (с галкой, `bg-accent`).
- Карта «Движок AI»: `ti-cpu`. Три выбираемые карточки: «CLI на подписке» (дефолт, рамка `2px --border-accent`, бейдж «дефолт»), «Свой ключ», «Ollama». Выбор переносит акцентную рамку.
- Карта «Выхлоп»: `ti-arrow-bar-to-down`. Чипы «Таблица .xlsx» / «Telegram-бот» + слайдер «Сопроводительное — порог» с живым `%`.
- Низ: «Сохранить» + акцентная «Запустить backfill».

**Экран 2 · Подборка:**
- Шапка прогона: «Подборка · <дата>» + «собрано N · после фильтра M · показаны топ-K» + кнопка `ti-download` «Скачать .xlsx».
- Карточки вакансий (по убыванию overall). Карточка: должность (15px/500); «компания · стадия · формат» (secondary); тег направления (если треков >1). Справа: подпись «резюме» + бейдж % (цвет по диапазону) + строка «карта X%» с `ti-map-2`. Ниже — строка вердикта (иконка по типу) + строка гэпа (muted). Кнопки: `ti-external-link` «Открыть», `ti-copy` «Сопроводительное» (**только если overall ≥ порога** — иначе кнопки нет), `ti-user-search` «Контакт + обращение».

## Расхождение с моделью tracks[] (обязательно учесть)

Прототип захардкодил **ровно два** направления в двух фиксированных колонках и заголовок «Профиль · под два пути». Это устаревшая модель. В реализации:
- Заголовок → «Профиль» (без «под два пути»).
- Две колонки → **повторяемая карточка направления** `[имя · резюме · шаблон · опц. рубрика/роли]` с «+ добавить направление» и удалением, минимум одна. Раскладка — та же сетка `auto-fit`, что в прототипе.
- Сабмит формы пишет `config.json` валидный по `config.schema.json` (массив `tracks[]` любой длины).
- N=1: карточка одна, теги направления в подборке/боте/xlsx скрыты.
