# Промты — индекс

Два слоя. **Пайплайновые** агент гоняет автоматически. **Setup** ты прогоняешь руками один раз, чтобы получить входы.

## Пайплайновые (агент, каждый прогон)

| Стадия | Файл | Тип | Вход | Выход |
|---|---|---|---|---|
| Нормализация | `normalize.md` | LLM | сырой пост | JSON `{title, company, link_or_contact, salary?, description}` |
| Пре-фильтр + роутинг | `prefilter-routing.md` | **код, не LLM** | резюме треков + вакансия (эмбеддинги) | `best_track` + финалисты |
| Скоринг | `scoring.md` | LLM (web) | резюме трека + рубрика + карта + вакансия + компания | строгий JSON (проценты, карта %, gaps, вердикт) |
| Сопроводительное | `cover-letter.md` | LLM | шаблон трека + резюме + вакансия + скоринг | текст письма (только overall ≥ порога) |
| Контакты (опц.) | `contact-search.md` | LLM (web) | роль + компания + регион | JSON: роли, запросы, кандидаты, черновик (без отправки) |

Сбор, дедуп, выхлоп — без промтов (скрап, хэши, запись файла).

## Setup (ты, один раз — рождает входы)

| Файл | Рождает | Куда |
|---|---|---|
| `setup/search-map-build.md` (6 промтов) | профиль+портрет → резюме → список компаний | `tracks[].rubric`, `tracks[].resume_path`, `search_map.path` |

## Что НЕ промт, хотя подаётся в промты

- **рубрика** (`tracks[].rubric`) и **карта поиска** (`search_map`) — входные данные, подставляются внутрь `scoring.md`. Рождаются setup-промтами.
- **шаблон сопроводительного** (`cover-templates/default.md`) — каркас для `cover-letter.md`.

## Связь с моделью tracks[]

Скоринг, сопроводительное и контакты работают против **одного направления**, выбранного роутингом. Резюме, рубрика и шаблон — у каждого трека свои; карта общая (примеры тегаются `track_id`). Захардкоженных путей нет.
