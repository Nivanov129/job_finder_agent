# Промт сопроводительного

Адаптировать **шаблон трека** под конкретную вакансию. Не генерить с нуля — это экономит токены и держит твой голос. Дефолтный шаблон — `cover-templates/default.md` (общий сопровод; по случаю адаптируем под интересные вакансии и выделяем релевантный опыт). Вызывается только когда `scores.overall >= cover_letter_threshold` и у трека есть `cover_template_path`.

## Вход

```
{{cover_template}}    — шаблон сопроводительного трека (структура и тон)
{{track_resume}}      — резюме трека (источник фактов: имя, должность, годы, технологии)
{{vacancy_text}}      — текст вакансии
{{company_name}}      — компания
{{vacancy_link}}      — ссылка (для мессенджеров)
{{scoring_result}}    — JSON скоринга (requirements.must, gaps.strategic, to_reach_100)
{{output_lang}}       — язык вывода
```

## Системная инструкция

возьми `{{cover_template}}` как каркас (структура, тон, длина). заполни плейсхолдеры из резюме и вакансии:
- `[название должности]`, `[X] годами опыта`, `[сфера]` — из `{{track_resume}}`;
- `[ссылка на вакансию]` — `{{vacancy_link}}` (только если уместно, для тг/мессенджеров);
- `[название из вакансии]`, `[технологии/задачи]` — зеркаль формулировки из `{{vacancy_text}}` и `requirements.must`;
- блок ценности (1–2 предложения) — подсвети из резюме то, что закрывает `requirements.must` и нейтрализует `gaps.strategic`, используя акценты из `to_reach_100`, но только на реальных фактах;
- финал — вопрос/призыв к действию (созвон).

не выдумывай опыт, которого нет в `{{track_resume}}`. не превышай длину шаблона существенно. убери воду. верни только текст письма на `{{output_lang}}`, без преамбулы и комментариев.
