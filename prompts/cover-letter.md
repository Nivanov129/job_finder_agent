# Промт сопроводительного

Адаптировать **шаблон трека** под конкретную вакансию. Не генерить с нуля — это экономит токены и держит твой голос. Дефолтный шаблон — `cover-templates/default.md` (общий сопровод; по случаю адаптируем под интересные вакансии и выделяем релевантный опыт). Вызывается только когда `scores.overall >= cover_letter_threshold` и у трека есть `cover_template_path`.

## Вход

```
{{cover_template}}    — приложенное сопроводительное: РЕЗУЛЬТАТЫ работы (достижения, метрики, кейсы) + структура и тон
{{track_resume}}      — резюме трека (источник фактов: имя, должность, годы, технологии)
{{vacancy_text}}      — текст вакансии
{{company_name}}      — компания
{{vacancy_link}}      — ссылка (для мессенджеров)
{{scoring_result}}    — JSON скоринга (requirements.must, gaps.strategic, to_reach_100)
{{output_lang}}       — язык вывода
```

## Системная инструкция

`{{cover_template}}` — не просто каркас: в нём твои РЕАЛЬНЫЕ результаты работы (достижения, метрики, кейсы). **Заходи именно с них.**

1. **Открывай письмо с 1–2 самых сильных и релевантных вакансии результатов** — конкретика «что сделал → какой эффект/метрика», а не общие слова. Бери их из `{{cover_template}}` (если там есть готовые результаты) и из `{{track_resume}}` (достижения, цифры, кейсы). Выбирай те, что прямее всего закрывают `requirements.must` вакансии. Если шаблон — это каркас с плейсхолдерами, результаты подставляй из резюме.
2. Дальше — короткий мост к вакансии: чем этот опыт полезен именно здесь. Зеркаль формулировки из `{{vacancy_text}}` и `requirements.must`, мягко закрой `gaps.strategic`, учти акценты `to_reach_100`.
3. Держи структуру, тон и длину `{{cover_template}}`. Заполни плейсхолдеры:
   - `[название должности]`, `[X] годами опыта`, `[сфера]` — из `{{track_resume}}`;
   - `[ссылка на вакансию]` — `{{vacancy_link}}` (только если уместно, для тг/мессенджеров);
   - `[название из вакансии]`, `[технологии/задачи]` — из `{{vacancy_text}}` и `requirements.must`.
4. Финал — короткий призыв к действию (созвон).

факты бери только из `{{cover_template}}` (результаты) и `{{track_resume}}` — ничего не выдумывай. не превышай длину шаблона существенно, убери воду. верни только текст письма на `{{output_lang}}`, без преамбулы и комментариев.
