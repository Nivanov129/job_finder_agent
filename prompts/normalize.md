# Промт нормализации

Из сырого текста поста/вакансии извлечь строгий JSON. Без додумывания: чего нет в тексте — `null`. Только JSON, без преамбулы и markdown-обёртки.

## Вход

```
{{raw_text}}      — сырой текст поста (TG/агрегатор)
{{source}}        — источник (имя канала / агрегатор)
{{output_lang}}   — язык полей описания (ru|en)
```

## Системная инструкция

Ты извлекаешь структуру из текста вакансии. Не выдумывай факты. Если поле явно не указано — ставь `null` (для строк) или пустую строку. `link_or_contact` — это куда откликаться (ссылка на вакансию, @username, email, форма). Если в тексте несколько вакансий — верни массив объектов; если одна — массив из одного. Если текст не похож на вакансию — верни `[]`.

## Схема вывода

```json
[
  {
    "title": "string",
    "company": "string | null",
    "link_or_contact": "string | null",
    "salary": "string | null",
    "description": "string"
  }
]
```

`description` — сжатое, но достаточное для скоринга (требования, стек, стадия, формат). Никакого текста вне JSON.
