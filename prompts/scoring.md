# Промт скоринга (7 этапов, роутинг по направлению)

Твой реальный промт «насколько % моё резюме подходит», адаптированный под пайплайн: работает против **одного** направления, которое уже выбрал пре-фильтр (лучший по косинусу), добавлен процент карты (хочу ли) и строгий JSON на выходе. При `multi_track_scoring=true` оркестратор зовёт промт по каждому из топ-2 треков и берёт результат с большим `overall`.

## Вход (подставляет оркестратор)

```
{{track_name}}            — выбранное направление (идёт в выхлоп как «направление»)
{{track_resume}}          — резюме под это направление
{{track_rubric}}          — «какая работа мне нужна» / портрет идеальной компании по этому направлению
{{track_disqualifiers}}   — дисквалификаторы трека + глобальные
{{search_map}}            — карта поиска (примеры идеальных вакансий), релевантные треку + общие
{{vacancy_text}}          — текст вакансии
{{company_name}}          — название компании
{{output_lang}}           — язык вывода (ru|en)
```

Доступен **web_search** — использовать только на этапе 1.

## Системная инструкция

я хочу оценить, насколько моё резюме соответствует конкретной вакансии с учётом контекста компании. не используй общие формулировки. не добавляй опыт, которого нет в резюме. будь объективен. вывод — строго JSON по схеме ниже, на языке `{{output_lang}}`, без преамбулы и markdown-обёртки.

## Этапы

**этап 1 — анализ компании** (web_search по `{{company_name}}`)
найди информацию о компании и проанализируй: размер (стартап / scaleup / корпорация); рынок (b2b / b2c / enterprise / tech / финтех и т.д.); стадию развития; продукт или тип бизнеса; предполагаемую культуру; уровень процессов (гибкость / бюрократия). если точных данных нет — делай осторожные логические выводы на основе публичной информации и формата вакансии. не фантазируй.

**этап 2 — структурный анализ вакансии**
выдели: обязательные требования (must have); желательные (nice to have); ключевые обязанности; уровень позиции; признаки seniority; контекстные ожидания (самостоятельность, масштаб, скорость).

**этап 3 — сопоставление с резюме**
сравни каждое обязательное и желательное требование с опытом из резюме. по каждому пункту: совпадает / частично / отсутствует. ничего не додумывай.

**этап 4 — расчёт соответствия**
рассчитай: % соответствия обязательным требованиям; % желательным; % уровню ответственности; % контексту компании; итоговый общий процент. кратко объясни метод расчёта. отдельно посчитай **% соответствия карте** — насколько вакансия похожа на эталоны из `{{search_map}}` и портрет из `{{track_rubric}}`. это «хочу ли туда», отдельно от «подхожу ли».

**этап 5 — gap-анализ**
раздели пробелы на: критичные (могут привести к отказу на скрининге); стратегические (влияют на финальные этапы); косметические. проверь по `{{track_disqualifiers}}` — сработавший дисквалификатор идёт в критичные и тянет overall вниз.

**этап 6 — как повысить процент до 100**
что усилить через переформулировку резюме; какие ключевые слова добавить; на чём сделать акцент; что реально прокачать; как нивелировать пробелы на интервью. без выдумывания фактов.

**этап 7 — финальный вывод**
стоит ли откликаться; это stretch-позиция или точное попадание; вероятность прохождения hr-скрининга; вероятность прохождения финального этапа.

## Чтение двух процентов

`overall` = подхожу ли под требования. `map_fit` = хочу ли туда / похоже ли на эталон. высокий `map_fit` при среднем `overall` → «дотянись, это твоё» (stretch); высокий `overall` при низком `map_fit` → «возьмут легко, но не то».

## Схема вывода (строгий JSON)

```json
{
  "track": "string — {{track_name}}",
  "company_analysis": "string",
  "company_confidence": "high | medium | low",
  "requirements": { "must": ["string"], "nice": ["string"] },
  "matching": [
    { "requirement": "string", "status": "full | partial | none", "evidence": "string" }
  ],
  "scores": {
    "must": 0,
    "nice": 0,
    "seniority": 0,
    "context": 0,
    "overall": 0,
    "map_fit": 0
  },
  "score_method": "string — краткое объяснение расчёта",
  "gaps": { "critical": ["string"], "strategic": ["string"], "cosmetic": ["string"] },
  "to_reach_100": ["string"],
  "verdict": {
    "should_apply": true,
    "type": "stretch | precise_fit",
    "hr_screening_probability": "high | medium | low",
    "final_stage_probability": "high | medium | low",
    "summary": "string — одна строка для карточки"
  }
}
```

все проценты — целые 0–100. никакого текста вне JSON.
