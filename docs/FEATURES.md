# Что работает — текущее состояние

Обзор рабочего функционала Job Agent. Продукт ставится локально (Docker), весь
скоринг идёт через твой AI (Claude Code / Codex CLI по подписке, либо бесплатные
OpenRouter / Ollama Cloud, либо API-ключ), данные не покидают машину, кроме как к
выбранному AI-движку и self-host web-поиску (SearXNG в составе стека).

## Веб-интерфейс

Основной способ работы — web-UI (по умолчанию `http://localhost:8766`). Левый
сайдбар + топбар, светлая дизайн-система, системные шрифты (без CDN). Экраны:

| Экран | Что делает |
|---|---|
| **Агент** | Дашборд always-on мониторинга: тумблер пуск/пауза, статы (найдено / топ ≥80% / оценено / письма), живая лента «что AI читает сейчас», карточка хоста, сильные совпадения. |
| **Подбор за период** | Разовый прогон за 24ч / 3 дня / неделю / месяц. Живая воронка: степпер (сбор → AI читает → фильтр → два процента) + прогресс + **лента реальных постов**. Галочка **«слать новые вакансии в Telegram-бот»**. |
| **Подборка** | Карточки финалистов с **двойными кольцами** (внешнее — карта %, внутреннее — резюме %), вердикт, гэп. **Накапливается в локальной БД** (`matches.db`) — растёт между прогонами и переживает рестарт; кнопка **«в архив»**. **Ползунок «совпадение по карте ≥»**, фильтр по направлению; кнопки «Открыть / Письмо / Контакт». Скачать `.xlsx`. |
| **Поиск контактов** | Контакты к произвольной вакансии (даже не из агента): **дай ссылку или загрузи PDF** — должность и компания определяются сами, дальше ищутся контакты + черновик обращения. Для `hh.ru` — через официальный API (страница за анти-ботом). |
| **Настройка** | Направления (резюме + карта + шаблон + авто-роли из резюме), источники (включая **карьерные сайты компаний**), порог сопроводительного, контакт-ассист, глубина backfill. |
| **AI · авторизация** | Выбор движка: **Claude Code** (токен подписки) / **Codex** (вход через ChatGPT) / **Ollama Cloud** / **OpenRouter** (бесплатные). Серверный логin или вставка токена/ключа, проверка. |
| **Telegram** | Вход в свой аккаунт (Telethon), авто-подбор каналов «про вакансии» + **бот для уведомлений** о новых вакансиях (подключение по кнопке). |

## Пайплайн подбора (стадии)

```
СБОР → дедуп → ФИЛЬТР ПО НАЗВАНИЮ → ПРЕ-ФИЛЬТР (эмбеддинги+роутинг) → СКОРИНГ → ОБОГАЩЕНИЕ → ВЫХЛОП
```

1. **Сбор** — параллельно из источников, каждый изолирован (падение одного не
   валит прогон): приватные/публичные Telegram-каналы (Telethon), агрегаторы
   `vseti.app` и `career.habr.com`. Опц. дорк-источники через web-поиск:
   **карьерные сайты компаний** (`career_sites`, напр. `career.ozon.ru` — поиск по
   ролям из резюме) и LinkedIn (`use_linkedin`, по умолчанию выкл).
2. **Грубый фильтр по названию должности** — из резюме AI выводит базовые
   названия ролей, отсекает заведомо нерелевантные посты ДО дорогого скоринга.
3. **Пре-фильтр + роутинг** — мультиязычные эмбеддинги (`paraphrase-multilingual-MiniLM (лёгкая, ~0.22 ГБ)`)
   считают близость резюме↔вакансия; пословный стем-устойчивый гейт по ролям
   (ловит русские варианты: «Менеджер по продукту» проходит роль «Менеджер
   продукта»); вакансия маршрутизируется в наиболее близкое направление.
4. **Скоринг** — AI даёт «два процента»: **резюме %** (подхожу ли) и **карта %**
   (хочу ли, против карты поиска), вердикт и гэпы. Вызовы AI параллельны.
5. **Обогащение** (опц., изолированно) — сопроводительное (выше порога) + контакты.

«Подбор за период» показывает **все** совпадения периода каждый раз (свежий
in-memory seen-store); **агент** использует постоянный seen-store, чтобы не
переоценивать/не дублировать между прогонами.

## Направления (`tracks[]`)

Динамический список 1..N. У каждого своё резюме, карта поиска, шаблон
сопроводительного и **«Допустимые роли», которые генерятся из резюме**
(кнопка «сгенерировать из резюме» + авто-подстановка после загрузки резюме —
AI достаёт базовые должности на языке резюме). При единственном треке теги
направления в UI скрываются.

## Контакты

Два режима, оба без отправки (только данные + черновик):

- **Основная выдача** (`find_contacts`) — быстрые булевы запросы по
  LinkedIn / Habr Career / Telegram / общему через SearXNG, заземление промта
  на выдачу, один вызов AI → кандидаты + черновик обращения.
- **Инвестигатор** (опц., `enable_contact_investigator`) — глубокое
  расследование: CLI-агент с встроенным web-поиском ведёт ветви по источникам,
  ищет по имени с транслитерацией, оценивает доказательства, ранжирует контакты
  с confidence. Выдача — отдельным блоком «с именем».

## Сопроводительное

Адаптирует приложенный шаблон под вакансию и **заходит с результатов работы**:
открывает 1–2 самыми сильными релевантными достижениями («что сделал → метрика»),
взятыми из шаблона/резюме под `requirements.must`. Генерится только выше
`cover_letter_threshold`.

## Движок и инфраструктура

- **Bring-your-own AI**: Claude Code / Codex (CLI на подписке), ключ
  Anthropic/OpenAI, или локальный Ollama. Переключение — правкой конфига.
- **Web-поиск**: self-host SearXNG в составе стека (`http://searxng:8080`);
  формат JSON включён, нужен для анализа компании и поиска контактов.
- **Тома**: модель эмбеддингов и seen-store на постоянных томах (переживают
  пересборку образа); каталог данных `/data` (config, .env, авторизация CLI).
- **Выхлоп**: «Подборка» в web-UI + таблица `.xlsx`. Цвет бейджей — единый
  (`presentation.py`), без дублирования.

## Telegram-бот (уведомления)

Помимо чтения каналов из своего аккаунта, можно подключить **отдельного бота**
(`@BotFather` → токен), чтобы получать **новые вакансии в личный чат**:

- подключение на странице «Telegram» — вставить токен, нажать Start у бота,
  «Подключить» (chat_id ловится через `getUpdates`); токен — в `.env`
  (`TELEGRAM_BOT_TOKEN`), chat_id — в конфиге (`owner_chat_id`);
- **агент-режим** шлёт новые вакансии (дедуп по seen-store) автоматически;
- **ручной «Подбор за период»** — по галочке «слать в Telegram-бот»;
- карточка = должность @ компания, проценты, вердикт, кнопки. Инвариант: только в
  личный чат владельца, без авто-DM кому-либо ещё.

## MCP-сервер (для Claude Desktop/Code)

Job Agent можно подключить как **MCP-сервер** к локальному агенту:
`uv run job-agent-mcp` (stdio, читает тот же `data/`). Инструменты:

- `list_matches` — подборка из локальной БД;
- `run_backfill` — запустить подбор за период;
- `find_contacts` — контакты к вакансии по ссылке / тексту / роль+компания.

## Безопасность

Данные не покидают машину, кроме как к выбранному AI и web-поиску. Никаких CDN,
телеметрии, автооткликов и авто-DM. Секреты (ключи, токены, Telethon-сессия) —
только через `.env`/каталог данных, в репозиторий не попадают (`/data/`
gitignored).
