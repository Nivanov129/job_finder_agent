# Job Agent — план реализации (ralphex)

Исполняемый план под ralphex. Каждая задача (`### Task N:`) — отдельная свежая сессия агента: атомарна, заканчивается чекбоксами и коммитом. После каждой задачи ralphex гоняет команды из `## Validation Commands`. Конвенции и инварианты — в `CLAUDE.md` (читается в начале каждой задачи). Этот файл — единственный источник правды; прозовый SCOPE он поглощает.

**Субстрат:** Python-код-пайплайн в Docker (не n8n) — см. «Решения».

## Validation Commands

- `uv run ruff check src tests`
- `uv run pytest -q`

## Overview

Локальный агент подбора вакансий. Ночью собирает вакансии из Telegram-каналов и агрегаторов, скорит под резюме пользователя его же AI (BYO), отдаёт `.xlsx` и/или дайджест в личный Telegram-бот. Ценность сверх hh: оценивает не только «подхожу ли» (резюме %), но и «хочу ли» (карта %), и готовит сопроводительное + контакт под отклик. Каждый участник ставит у себя.

## Context

Раздел справочный — без чекбоксов. Читать перед каждой задачей.

### Жёсткие рамки (non-goals)

- Всё локально на машине пользователя. Ничего не хостится централизованно, ключи не раздаются.
- Bring-your-own AI: пользователь подключает свой Claude/Codex/ключ/Ollama. Продукт бесплатный.
- Без автооткликов. Без авто-отправки сообщений (контакт-модуль готовит черновик, отправка руками).
- Без парсинга hh.ru и LinkedIn (hh пользователь мониторит сам; LinkedIn враждебен к автоматизации).
- Данные не покидают машину, кроме как к выбранной пользователем AI.
- Always-on: ночной мониторинг требует включённого хоста; backfill — по требованию.

### Модель направлений (tracks) — ядро универсальности

Захардкоженных «двух путей» нет. Есть расширяемый список `tracks[]` от 1 до N. Каждый трек самодостаточен: своё резюме, шаблон сопроводительного, рубрика, опц. гейт по роли и дисквалификаторы. Кейс N=1 схлопывает UI и выхлоп (тег направления и колонка скрываются), логика не ветвится. Схема — `config.schema.json`, пример — `config.example.json`.

### Пайплайн (стадии)

1. **Сбор.** Адаптеры: TG публичные (`t.me/s/<канал>`), TG приватные (Telethon, опц.), vseti.app, getmatch. Выход: `{raw_text, source, url, date}`.
2. **Нормализация.** AI по `prompts/normalize.md`: из сырого текста → `{title, company, link_or_contact, salary?, description}`.
3. **Дедуп.** Против SQLite-хранилища виденных (хэш `title+company` + url), кросс-источник.
4. **Пре-фильтр + роутинг.** Локальные эмбеддинги: косинус резюме каждого трека ↔ вакансия → `best_track`; гейт по роли; порог близости. Отсечь до ~15–25 финалистов. Спека — `prompts/prefilter-routing.md`. Без фильтра по зарплате.
5. **Скоринг.** 7-этапный промт `prompts/scoring.md` по финалистам против выбранного трека, web-поиск на анализе компании. Выход — строгий JSON (проценты резюме, карта %, gap-анализ, вердикт, направление).
6. **Обогащение** (overall ≥ порога): сопроводительное (`prompts/cover-letter.md`, правка шаблона трека); контакты (опц., `prompts/contact-search.md`, без отправки).
7. **Выход.** `.xlsx` и/или карточки в бот. Обновить хранилище виденных.

Два режима поверх стадий: **backfill** (ручной, история за N дней) и **nightly** (по расписанию, инкремент).

### Форматы выхлопа

- **XLSX**, колонки: `дата · источник · должность · компания · ссылка/контакт · зарплата · резюме % · карта % · направление · вердикт · гэпы · сопроводительное · контакты`. Колонка `направление` скрыта при единственном треке.
- **Карточка бота**: `должность @ компания` · тег направления (если треков >1) · бейдж «резюме %» (зелёный/янтарный/серый по диапазону) · «карта %» · строка вердикта · строка гэпа · кнопки «Открыть», «Скопировать сопроводительное» (если выше порога), «Контакт + обращение». Топ по overall за прогон.

### Чтение двух процентов

`overall` (резюме %) = подхожу ли под требования. `map_fit` (карта %) = хочу ли туда. Высокая карта при средней резюме → «дотянись, это твоё» (stretch); высокая резюме при низкой карте → «возьмут легко, но не то».

### Источник дизайна

Визуальный язык — `design/design-tokens.md` (извлечён из `design/prototype.html`): палитра, бордеры 0.5px, радиусы (карточки 12px), бейджи процентов по диапазонам, иконки вердикта, инвентарь обоих экранов. Прототип показывает **устаревшую** захардкоженную пару путей — структуру UI обобщать под `tracks[]` (раздел «Расхождение» в токенах). Иконки Tabler вшивать локально, не с CDN. Бейдж-диапазоны процентов общие для xlsx, бота и web-UI (см. Task 0.4).

### Решения (не перерешивать)

| Вопрос | Решение |
|---|---|
| Субстрат | Python-код-пайплайн в Docker. n8n отклонён: его «валидация» — визуальный JSON-граф, неподходящий для ralphex (нет `pytest`/`ruff`), а тяжёлая логика в n8n всё равно живёт во внешнем коде. |
| Источник hh | Исключён (мониторится вручную). |
| Источники | TG-каналы (публичные `t.me/s/`, приватные Telethon опц.) + vseti.app + getmatch. Карьерные страницы компаний — вторая волна. |
| Движок скоринга | BYO: `cli` (Claude Code/Codex, дефолт) · `api_key` (Anthropic/OpenAI) · `ollama` (локально). |
| Эмбеддинги пре-фильтра | Фиксированная локальная модель (`bge-m3` через fastembed), НЕ зависит от выбора `scoring_engine`. |
| Web-поиск | SearXNG self-host или SERP-ключ. |
| Выхлоп | `.xlsx` + дайджест в личный бот. Backfill (14 дней) → ночной инкремент. |
| Сопроводительное | Правка шаблона трека (не генерим с нуля), только overall ≥ порога (дефолт 70). |
| Карта поиска | Общий файл; примеры можно тегать `track_id`. |
| Контакты | Опц., по умолчанию выкл. Находим + черновик, отправки нет. |
| Мульти-трек скоринг | Дефолт выкл (роутинг). Опц. скорить топ-2 трека в пределах `multi_track_delta`. |
| Распространение | `compose.yml` + тонкий install-скрипт. Не нативный bundle с вшитым докером. |

Открытый калибровочный момент: минимальный порог `best_sim` в пре-фильтре числом не зафиксирован — калибруется на backfill (Task 4.4). До тех пор — мягкий дефолт, конфигурируемый.

## Риски

- Никаких автооткликов и авто-DM (бан аккаунтов + репутация). Бот пишет только в личный чат владельца.
- Никакого парсинга hh/LinkedIn.
- Резюме не лить в API-тарифы, обучающиеся на промтах: предпочесть локаль/CLI или no-training-провайдера (в README).
- Скрап агрегаторов хрупкий — логика изолирована в адаптерах, поломка одного не валит прогон.
- `cli`-путь жжёт агентный лимит подписки; переключение на ключ/Ollama — только конфиг, без правки кода.
